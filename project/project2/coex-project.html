Problem & Resolve

pc와 mobil의 내용이 다르다!

-해결법-

! 모바일은 pc버전의 화면과 다르게 크기에 따라 숨겨지는 글들이 있다 ! 이때 span, strong 등을 사용해서
p사이에 class나 id 값을 주고 css에서 display: none 또는 display: block을 사용해 보이거나 숨겼다.

크기가 고정된 구조의 화면과 다르게 반응형 웹은 각 화면 크기 버전에 따라 html 관리가 필요하다!

(px로 사이즈를 조절하면 grid나 flex, position의 레이아웃이 흘러넘칠때가 있다.)

- 해결법 -

크기 고정값이 방해되는 부분을 %로 변경

색션과 이미지를 %와 px을 섞어가며 배치, min-width, max-width를 사용해 일정 이상 커지지 않게 관리하여 흘러넘침 해결 position로 고정이 필요한 부분에 적용

-> 반응형임을 고려해 html 문서 시작부터 작성하고 좀 더 효율적이고 짧은 코드로 관리할 수 있는 방법을 찾아 다음 프로젝트에서 적용해 볼 예정

프로젝트 완료 리뷰

이미지나 화면을 고정값으로 쉽게 제어할 수 있던 일반 웹사이트와 다르게 반응형은 각 요소의 크기와 배치가 달라져 처음 구상했던 구조와는 다르게 새로 고쳐야 할 일이 많았다.

pc버전에서만 보여지는 글은 div와 span을 사용해 숨김 기능이 가능하게 수정하였고, 미디어 쿼리 수업 내용을 참고하여 화면 크기당 달라지는 여백과 이미지들은 px에서 %로 값으로 변경했다.
이 작업을 통해 중요한 내용의 요소들이 어떡해야 자주 사람들에게 효과적으로 눈에 들어올지 고민하며 사용자를 위한 페이지란 무엇인가, 가독성을 위해서 색이나 크기 등은
어떻게 배치해야 하는가를 생각하는 시간이었다.

이렇듯 사용자의 편의성이 중요한 프로젝트라 시각적인 css 효과에 대해 많은 공부가 필요했다. 지금 보았을때 보완할점이 눈에 띄게 많아 아쉽지만 다음 작업에서 조금 더 효율적인 레이아웃을 짜고 작업 방식으로 진행하겠다는 다짐과 함께 마무리 지었다.
